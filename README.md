# 데이터 조회 연계 예제 프로젝트
## 프로젝트 개요
외부에서 관리하는 학생의 출석 데이터를 받아 상태를 화면에 표시하고 조회 할 수 있는 예제입니다.

## 실행 방법
1. 모든 파일을 같은 폴더 안에 저장한다.
2. index.html을 웹 브라우저로 연다.
3. 상단 필터 버튼을 클릭하여 상태별로 학생을 조회한다.

## 주요 기능
- 외부에서 관리하는 학생 데이터를 이용하여 카드 목록으로 표시한다.
- 필터를 이용하여 출석, 지각, 결석을 확인할 수 있다.
- 학생의 이름, 반, 상태를 확인할 수 있다.
- 상태를 색으로 구분할 수 있다.
  - 출석: 녹색
  - 지각: 노랑
  - 결석: 빨강

## 시험 요구사항 설명

### 외부 서버는 무엇인가?
외부 서버란 학생과 같은 핵심 데이터를 프론트엔드와 분리하여 관리하는 공간을 의미합니다. 본 프로젝트에서는 학생의 이름, 반, 출석 상태 정보를 외부 서버에서 전달받는 데이터로 가정하여 해당 데이터를 이용해 학생 목록을 카드 형태로 화면에 표시합니다. 외부 서버는 자신이 관리하는 학생 출석 데이터 내부 시스템에 제공하는 역할을 담당합니다.

### 어떤 데이터가 연계 데이터인가?
본 프로젝트에서 연계 데이터란 외부 서버에서 관리되며 화면에 표시되는 학생 정보 데이터를 의미합니다. 학생의 학번, 이름, 반, 출석 상태 정보가 서로 연결되어 하나의 학생 데이터로 구성되어 이 데이터를 기반으로 외부 시스템으로 수신하는 과정을 의미합니다.

### 외부 서버에서 내부 데이터 흐름
첫 번째, 외부 서버에서는 학생의 학번, 이름, 반, 출석 상태와 같은 학생 정보를 관리합니다.
두 번째, 내부 웹 애플리케이션은 외부 서버로부터 전달받은 학생 데이터를 기반으로 화면을 구성합니다.
이후 전달받은 데이터는 카드 목록 형태로 가공되어 사용자에게 시각적으로 제공되며 이를 통해 외부 서버와 내부 웹 간의 데이터 연계 흐름을 이해할 수 있습니다.

### 왜 이 구조가 연계 구조일까?
본 프로젝트의 구조는 외부 서버의 데이터와 내부 웹 화면이 연결되어 동작하기 때문에 연계 구조라고 할 수 있습니다. 외부 서버에서 관리되는 학생 데이터는 내부 웹 애플리케이션으로 전달되어 카드 목록 생성에 사용됩니다. 내부 웹에서는 전달받은 데이터를 기준으로 출석, 지각, 결석 상태를 필터링합니다. 그 결과에 따라 화면에 표시되는 카드 목록이 즉시 변경됩니다. 이처럼 데이터 처리와 화면 반영이 서로 연계되어 동작하는 구조입니다.

### 이 예제를 통한 평가 가능한 능력은 뭘까?
본 예제를 통해 외부 데이터 구조를 이해하고 이를 화면에 출력하는 능력을 평가할 수 있습니다. 또한 JavaScript 배열을 활용하여 데이터를 가공하고 관리하는 능력과 DOM 조작을 통해 동적으로 UI를 생성하고 제어하는 능력을 확인할 수 있습니다. 학생 상태와 같은 필터 기능 구현을 통해 사용자 입력에 따른 데이터 처리 능력을 평가할 수 있습니다. 더 나아가 데이터 상태 변화에 따라 화면을 즉시 갱신하는 로직 설계 능력을 확인할 수 있습니다. 종합적으로 프론트엔드에서 데이터 흐름과 화면 연계 구조를 이해하는 능력을 평가할 수 있습니다.
